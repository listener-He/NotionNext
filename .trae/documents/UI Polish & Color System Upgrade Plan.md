# 用户界面优化与配色系统升级计划

我将根据以下具体修改来回应用户的反馈：

## 1. 视觉修复与优化

* **博客帖子卡片（`BlogPostCard.js`）**：
  **问题**：文本区域过窄。
  **修复**：调整弹性比例。目前图片的弹性比例为`w-full md:w-5/12`，文本的弹性比例为`flex-1`。 我需要查看文本为什么没有把右边空白区域填充上 同时我也要为图片宽高比设置标准的样式。
* **目录进度条（`Progress.js`** **&** **`style.js`）**：
  **问题**：无填充颜色，缺乏视觉反馈。
  **修复**：更新`themes/hexo/style.js`，为`.reading-progress-bar`定义一个特定的渐变类。
  \* **日间模式**：柔和的“暖色-清新系”渐变。
  \* **夜间模式**：采用“赛博”渐变色（深空青色 -> 蓝色 -> 紫色），在深色背景下提供更好的可见性。
* **打字机字体（`Hero.js`）**：
  **问题**：当前字体难看。
  **修复**：将`#typed`元素应用其他更美观度高的系统字体 并设计观赏性好的字体样式

## 2. 颜色系统升级（`tailwind.config.js`）

* **现代渐变**：
  * 添加多色渐变，营造“轻奢”氛围。
  * **日间**：`bg-gradient-to-r from-indigo-50 via-white to-indigo-50` -> 升级为 `from-neutral-50 via-slate-50 to-zinc-50`，并添加微妙的金色点缀。
  * **夜间**：`from-gray-900 to-black` -> 升级为 `from-slate-900 via-indigo-950 to-slate-900`。
* **进度条颜色**：为进度条渐变定义自定义工具。

## 3. 错误修复验证

* 上一个版本的bug修复（LazyImage 渲染、Algolia 钩子）已经实现。我将确保不会出现任何退步。
